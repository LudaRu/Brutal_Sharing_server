# AUTH компонент 

Порядок авторизации.

1.  Клиент: /auth/session

    Сервер: Отдает клиенту временный идентификатор.
        для связывания утентификации по Oauth2 (т.к. CROSS запрос)     
   
2.  Клиент: /auth/vk?id=SESSION_ID

    Сервер: Перенаправляет на vk для авторизации
    
3.  Клиент: редиректиться -> /auth/vk/callback?id=SESSION_ID....

    Сервер: Сохраняет и связывает результат с SESSION_ID
       
4.  Клиент: Возвращается в свой клиент.
        Делает /auth/vk/SESSION_ID - Получает JWT токен аутификации для REST запросов
        
    Сервер: Отдает JWT токен 
   
   
4.  Клиент: Делает REST запросы с header -> x-auth-token=JWT_TOKEN

    Сервер: Аутифицируем JWT_TOKEN и тд...
